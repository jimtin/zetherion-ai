# API Auth Matrix

Authentication and authorization matrix for both API surfaces.

Authoritative exposure rule:
- External clients authenticate only against CGS `/service/ai/v1`.
- Zetherion `/api/v1` auth is internal upstream auth between CGS and Zetherion.

## Maintenance Note (2026-03-04)

- Added token-authenticated CGS blog publish adapter route:
  - `POST /service/ai/v1/internal/blog/publish`
- Internal lifecycle tenant mutations now enforce claims-to-tenant policy in addition to operator scope checks.
- Auth requirements for public document routes are unchanged.

## Zetherion Upstream API (`/api/v1`, internal)

| Endpoint Group | Auth Mode | Header | Tenant Scope Source |
|---|---|---|---|
| Health (`GET /api/v1/health`) | none | none | none |
| Sessions (`/sessions`) | API key | `X-API-Key` | tenant resolved from API key |
| Chat + analytics session routes | session bearer | `Authorization: Bearer zt_sess_...` | tenant/session resolved from token claims |
| CRM reads + release markers | API key | `X-API-Key` | tenant resolved from API key |
| Documents + RAG + model catalog | API key | `X-API-Key` | tenant resolved from API key |

Notes:
- Session tokens are generated by `POST /api/v1/sessions` and are short-lived tenant-scoped JWTs.
- Document routes do not accept session bearer tokens; they require tenant API key auth.

## CGS Gateway (`/service/ai/v1`, public)

| Endpoint Group | Auth Mode | Header | Tenant Enforcement |
|---|---|---|---|
| Health (`GET /service/ai/v1/health`) | none | none | none |
| Runtime conversations + analytics + recommendations | CGS bearer JWT | `Authorization: Bearer <cgs_jwt>` | conversation mapping + principal tenant check |
| Document + RAG + provider catalog routes | CGS bearer JWT | `Authorization: Bearer <cgs_jwt>` | explicit `tenant_id` request/query + principal tenant check |
| Internal operator routes (`/service/ai/v1/internal/*`) | CGS bearer JWT + role/scope | `Authorization: Bearer <cgs_jwt>` | requires operator/admin scope |
| Blog publish adapter (`/service/ai/v1/internal/blog/publish`) | static bearer token | `Authorization: Bearer <CGS_BLOG_PUBLISH_TOKEN>` + `Idempotency-Key` | no JWT principal; token + `blog-<sha>` contract required |
| Internal tenant-admin routes (`/service/ai/v1/internal/admin/tenants/*`) | CGS bearer JWT + operator scope + admin scope + step-up (mutations) | `Authorization: Bearer <cgs_jwt>` | tenant path must be authorized; secrets routes require `cgs:zetherion-secrets-admin` |
| Internal tenant email admin routes (`/service/ai/v1/internal/admin/tenants/*/email/*`) | CGS bearer JWT + operator scope + admin scope + step-up (mutations) | `Authorization: Bearer <cgs_jwt>` | oauth app read/write require `cgs:zetherion-secrets-admin`; mailbox disconnect requires approved `change_ticket_id` |
| Reporting routes (`/service/ai/v1/tenants/*`) | CGS bearer JWT | `Authorization: Bearer <cgs_jwt>` | path tenant must match principal tenant unless operator privileges |

## Upstream Credential Mapping (Gateway -> Zetherion)

| CGS Route Type | Upstream Credential Used |
|---|---|
| Public API key routes (sessions, docs, RAG, providers, CRM, release markers) | `X-API-Key: <mapped tenant key>` |
| Session-scoped routes (chat, analytics, recommendations) | `Authorization: Bearer <mapped zt_sess_ token>` |
| Internal tenant lifecycle skills intents | `X-API-Secret` between gateway and skills API |
| Internal tenant-admin skills REST routes | `X-API-Secret` + signed actor envelope (`X-Admin-Actor`, `X-Admin-Signature`) |

Tenant-claim policy note:
- Internal lifecycle tenant mutations (`/service/ai/v1/internal/tenants/{tenant_id}/*`) enforce operator tenant claim checks (`tenant_id`/`allowed_tenants`) in addition to operator scope.

## Browser Upload/Download UX Support (via CGS)

After CGS maps these routes, browser clients can implement:
- upload flow via `/service/ai/v1/documents/uploads` + `/complete`
- inline preview via `/service/ai/v1/documents/{document_id}/preview`
- file download via `/service/ai/v1/documents/{document_id}/download`

CGS keeps tenant/session secrets server-side; browser clients never receive Zetherion API keys.
