    Key Design Patterns¶

    1. Factory Pattern - Router backend selection (Gemini vs Ollama)
    2. Strategy Pattern - Pluggable LLM backends
    3. Repository Pattern - Memory abstraction layer
    4. Singleton Pattern - Configuration management with LRU cache
    5. Async/Await - Non-blocking I/O throughout the stack

    Data Flow¶

    User Message (Discord)
  ↓

  ↓

  ↓
┌─────────────────┬─────────────────┐

  ↓

  ↓

  ↓
Response Splitting (2000 char Discord limit)
  ↓
User Response (Discord)

    Development Environment¶

    Prerequisites¶

    • Python: 3.12 or higher (tested on 3.12 and 3.13)
    • Docker Desktop: Latest version (for Qdrant and Ollama)
    • Git: 2.30+
    • Code Editor: VSCode recommended (see .vscode/settings.json)

    Initial Setup¶

    1. Clone and set up remote: 

      git clone https://github.com/jimtin/zetherion-ai.git
cd zetherion_ai
git remote add upstream https://github.com/jimtin/zetherion-ai.git

    2. Create virtual environment: 

      python3.12 -m venv venv
source venv/bin/activate  # macOS/Linux
# OR
venv\Scripts\activate  # Windows

    3. Install dependencies: 

      pip install --upgrade pip
pip install -r requirements.txt
pip install -r requirements-dev.txt

    4. Install pre-commit hooks: 

      pre-commit install
pre-commit install --hook-type pre-push

    5. Set up environment variables: 

      cp .env.example .env
# Edit .env with your API keys

    6. Verify setup: 

      python -c "from zetherion_ai.config import get_settings; print('Config loaded successfully')"
pytest tests/ -m "not integration and not discord_e2e" --maxfail=1

    Development Workflow¶

    Feature Development Cycle¶

    1. Create feature branch: 

      git checkout -b feature/my-new-feature

    2. Write failing tests first (TDD): 

      # Create test file
touch tests/test_my_feature.py

# Write test
# Run tests (should fail)
pytest tests/test_my_feature.py -v

    3. Implement feature: 

      # Create/modify source files
# Run tests iteratively
pytest tests/test_my_feature.py -v --maxfail=1

    4. Verify all tests pass: 

      pytest tests/ -m "not discord_e2e" --cov=src/zetherion_ai

    5. Check code quality: 

      ruff check --fix .
ruff format .
mypy src/zetherion_ai

    6. Commit with conventional format: 

      git add .
git commit -m "feat: add amazing new feature"
# Pre-commit hooks run automatically

    7. Push and create PR: 

      git push origin feature/my-new-feature
# Create PR on GitHub

    Branch Strategy¶

    • main - Production-ready code (protected)
    • feature/* - New features
    • fix/* - Bug fixes
    • docs/* - Documentation updates
    • test/* - Test improvements
    • refactor/* - Code refactoring

    Pre-Commit Workflow¶

    When you commit, the following hooks run automatically:

    1. File Checks (100ms)
    2. Trailing whitespace removal
    3. End-of-file fixer
    4. Merge conflict markers

    5. Large file prevention

    6. Gitleaks (1-2s)

    7. Secret scanning (12 custom rules)

    8. Zero false positives (tuned allowlists)

    9. Ruff Linting (1-2s)

    10. 600+ lint rules
    11. Auto-fixes applied

    12. Import sorting

    13. Ruff Formatting (0.5s)

    14. PEP 8 compliance

    15. 100-char line length

    16. Mypy Type Checking (3-5s)

    17. Strict mode

    18. All source files

    19. Bandit Security Scan (2-3s)

    20. Python security issues

    21. OWASP top 10 checks

    22. Hadolint (0.5s)

    23. Dockerfile best practices

    Total pre-commit time: ~10-15 seconds

    Pre-Push Workflow¶

    Before pushing, additional checks run:

    1. Ruff (full project)
    2. Mypy (full project)
    3. Pytest (all tests with coverage)

    Total pre-push time: ~30-60 seconds

    Performance Optimization¶

    Critical Optimizations Applied¶

    1. Async Qdrant Client - Prevents event loop blocking 

      # ❌ Bad (blocks event loop)
from qdrant_client import QdrantClient
client = QdrantClient(url="http://qdrant:6333")

# ✅ Good (non-blocking)
from qdrant_client import AsyncQdrantClient
client = AsyncQdrantClient(url="http://qdrant:6333")

    2. Parallel Embeddings - 10x faster for batches 

      # ❌ Bad (sequential)
embeddings = [await embed_text(text) for text in texts]

# ✅ Good (parallel)
tasks = [embed_text(text) for text in texts]
embeddings = await asyncio.gather(*tasks)

    3. Deduplicated Memory Searches - 50% fewer API calls 

      # ❌ Bad (searches twice)
context_claude = await search_memory(message)
context_gemini = await search_memory(message)

# ✅ Good (searches once)
context = await search_memory(message)
response_claude = await generate_claude(message, context)
response_gemini = await generate_gemini(message, context)

    4. Retry with Exponential Backoff - Handles transient failures 

      retries = 0
while retries < max_retries:
    try:
        return await api_call()
    except TransientError:
        await asyncio.sleep(2 ** retries)
        retries += 1

    Docker Development¶

    Development vs Production Compose¶

    Development (docker-compose.dev.yml): - Hot-reload for code changes - Volume mounts for local development - Exposed ports for debugging

    Production (docker-compose.yml): - Optimized image layers - Health checks - Restart policies - Resource limits (optional)

    Common Docker Tasks¶

    Start development environment: 

    docker compose -f docker-compose.dev.yml up -d

    Rebuild after dependency changes: 

    docker compose build --no-cache
docker compose up -d

    View logs: 

    # All services
docker compose logs -f

# Specific service
docker compose logs zetherion_ai -f

# Last 100 lines
docker compose logs --tail=100 zetherion_ai

    Restart single service: 

    docker compose restart zetherion_ai

    Clean up everything: 

    docker compose down -v  # Removes volumes (data loss!)
docker system prune -a  # Removes all unused images

    1. Check latest models:
    2. Claude: https://docs.anthropic.com/en/docs/about-claude/models
    3. OpenAI: https://platform.openai.com/docs/models

    4. Gemini: https://ai.google.dev/gemini-api/docs/models

    5. Update src/zetherion_ai/config.py: 

      claude_model: str = Field(default="claude-sonnet-4-5-20250929")
openai_model: str = Field(default="gpt-4o")
router_model: str = Field(default="gemini-2.5-flash")

    6. Test new models: 

      pytest tests/integration/test_e2e.py -v

    7. Update documentation (comment in config.py with date)

    Update Dependencies¶

    Check for updates: 

    pip list --outdated

    Update specific package: 

    pip install --upgrade package-name
pip freeze | grep package-name >> requirements.txt

    Update all dev dependencies: 

    pip install --upgrade -r requirements-dev.txt
pip freeze > requirements-dev.txt

    Test after updates: 

    pytest tests/ -m "not discord_e2e"

    Troubleshooting¶

    Common Development Issues¶

    Issue: Pre-commit hooks fail 

    # Fix Ruff issues
ruff check --fix .
ruff format .

# Fix Mypy issues
mypy src/zetherion_ai --show-error-codes

# Skip hooks (emergency only)
git commit --no-verify

    Issue: Tests fail with import errors 

    # Ensure virtual environment is activated
source venv/bin/activate

# Reinstall in editable mode
pip install -e .

    Issue: Docker services won't start 

    # Check Docker daemon
docker info

# Check ports are free
lsof -i :6333  # Qdrant
lsof -i :11434  # Ollama

# Restart Docker Desktop
# macOS: osascript -e 'quit app "Docker"' && open -a Docker

    Issue: Ollama model not found 

    docker exec zetherion_ai-ollama ollama list
docker exec zetherion_ai-ollama ollama pull llama3.1:8b

    Issue: Qdrant connection refused 

    # Check Qdrant is running
docker ps | grep qdrant

# Check health
curl http://localhost:6333/health

# Restart Qdrant
docker compose restart qdrant
